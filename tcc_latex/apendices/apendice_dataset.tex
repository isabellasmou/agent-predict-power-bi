\chapter{Dataset do Experimento Controlado}
\label{ap:dataset}

Este apêndice apresenta o dataset completo dos 32 casos de teste utilizados no experimento controlado (OE6), incluindo expressões originais, gabaritos e resultados obtidos, distribuídos em duas fases de avaliação (Capítulo 5).

\section{Estrutura do Dataset}

O dataset é gerado e persistido em formato JSON, com os seguintes campos por caso de teste:

\begin{itemize}
    \item \textbf{caso\_id}: Identificador do caso de teste (ex: D01, RT02, RM05, DC03, DT01)
    \item \textbf{tipo\_mudanca}: Tipo de mudança estrutural (rename\_column, rename\_table, rename\_measure, delete\_column, delete\_table)
    \item \textbf{descricao}: Descrição textual do caso, incluindo o objeto alvo e a medida testada
    \item \textbf{fase1\_sucesso}: Booleano indicando se a Fase 1 (análise de impacto) identificou corretamente o objeto esperado
    \item \textbf{fase1\_objetos\_encontrados}: Lista de todos os objetos impactados retornados pelo \texttt{ImpactAnalyzer}
    \item \textbf{fase1\_requires\_manual\_review}: Booleano indicando se o agente sinalizou necessidade de revisão manual
    \item \textbf{fase2\_sucesso}: Booleano indicando se a Fase 2 (aplicação) produziu o resultado correto
    \item \textbf{fase2\_expressao\_gerada}: Expressão DAX gerada pelo LLM (para renomeios) ou \texttt{null} (para deleções bloqueadas)
    \item \textbf{fase2\_tempo\_segundos}: Tempo de resposta do LLM em segundos (renomeios) ou 0.0 (deleções)
\end{itemize}

\section{Casos de Teste por Tipo de Mudança}

Todos os casos foram executados sobre o modelo semântico real descrito na Seção~\ref{sec:modelo-real} (pipeline de CI/CD Jenkins/Azure DevOps, 12 tabelas, 100 medidas). Por concisão, são apresentados a seguir exemplos representativos de cada tipo; o dataset completo com os 32 casos está disponível no repositório do projeto (Seção~\ref{sec:disponibilidade-dataset}).

\subsection{Renomeio de Coluna (8 casos)}

\subsubsection{Caso D01: Coluna Plataforma, impacto em cascata de 54 objetos}

\begin{itemize}
    \item \textbf{Mudança}: Renomear \texttt{dGeral[Plataforma]} $\rightarrow$ \texttt{dGeral[PlataformaUpdated]}
    \item \textbf{Medida testada}: \texttt{Mediana Segundos}
    \item \textbf{Fase 1}: 54 objetos impactados identificados corretamente, incluindo dependências em cascata (ex: \texttt{Mediana de Pipeline Execução}, que depende de \texttt{Mediana Segundos})
    \item \textbf{Fase 2 -- Expressão Original (trecho)}:
\end{itemize}

\begin{verbatim}
SWITCH(
 TRUE(),
 SELECTEDVALUE(dGeral[Plataforma]) = "Jenkins",
 ...
)
\end{verbatim}

\begin{itemize}
    \item \textbf{Fase 2 -- Expressão Gerada}: idêntica ao gabarito, com todas as ocorrências de \texttt{dGeral[Plataforma]} substituídas por \texttt{dGeral[PlataformaUpdated]}
    \item \textbf{Resultado}: Correto (Fase 1 e Fase 2)
    \item \textbf{Tempo}: 1,84s
\end{itemize}

\subsubsection{Caso D03: Coluna Status Pipe Alterado}

\begin{itemize}
    \item \textbf{Mudança}: Renomear \texttt{dGeral[Status Pipe Alterado]} $\rightarrow$ \texttt{dGeral[Status Pipe AlteradoUpdated]}
    \item \textbf{Medida testada}: \texttt{Mediana Segundos}
    \item \textbf{Fase 1}: 13 objetos impactados identificados corretamente
    \item \textbf{Resultado}: Correto (Fase 1 e Fase 2)
    \item \textbf{Tempo}: 20,11s (um dos três casos com maior latência, atribuída à API externa e não à complexidade da expressão)
\end{itemize}

\subsection{Renomeio de Tabela (4 casos)}

\subsubsection{Caso RT01: Tabela dGeral, impacto em 89 objetos}

\begin{itemize}
    \item \textbf{Mudança}: Renomear tabela \texttt{dGeral} $\rightarrow$ \texttt{dGeralUpdated}
    \item \textbf{Medida testada}: \texttt{Projetos Jenkins}
    \item \textbf{Fase 1}: 89 objetos impactados identificados corretamente, o maior número de dependências em cascata do experimento
    \item \textbf{Fase 2 -- Expressão Gerada (trecho)}:
\end{itemize}

\begin{verbatim}
CALCULATE(
 DISTINCTCOUNT('dGeralUpdated'[Nome Projeto]),
 'dGeralUpdated'[Plataforma] = "Jenkins",
 NOT 'dGeralUpdated'[Nome Projeto] IN ExcludeList
)
\end{verbatim}

\begin{itemize}
    \item \textbf{Resultado}: Correto (Fase 1 e Fase 2)
    \item \textbf{Tempo}: 1,12s
\end{itemize}

\subsection{Renomeio de Medida (8 casos)}

\subsubsection{Caso RM01: Medida Execuções Sucesso}

\begin{itemize}
    \item \textbf{Mudança}: Renomear medida \texttt{[Execuções Sucesso]} $\rightarrow$ \texttt{[Execuções Sucesso Updated]}
    \item \textbf{Medida testada}: \texttt{Porcentagem Sucesso}
    \item \textbf{Expressão Original}: \texttt{DIVIDE([Execuções Sucesso], [Total de Execuções], BLANK())}
    \item \textbf{Expressão Gerada}: \texttt{DIVIDE([Execuções Sucesso Updated], [Total de Execuções], BLANK())}
    \item \textbf{Resultado}: Correto (Fase 1 e Fase 2)
    \item \textbf{Tempo}: 2,42s
\end{itemize}

\subsection{Deleção de Coluna (8 casos)}

\subsubsection{Caso DC02: Deleção da coluna Plataforma}

\begin{itemize}
    \item \textbf{Mudança}: Deletar \texttt{dGeral[Plataforma]}
    \item \textbf{Medida testada}: \texttt{Projetos Jenkins}
    \item \textbf{Fase 1}: 54 objetos impactados identificados, com \texttt{requires\_manual\_review=True}
    \item \textbf{Fase 2}: aplicação automática corretamente bloqueada -- o agente não tentou gerar uma correção, sinalizando a dependência ativa para revisão manual
    \item \textbf{Resultado}: Correto (bloqueio esperado)
\end{itemize}

\subsection{Deleção de Tabela (4 casos)}

\subsubsection{Caso DT01: Deleção da tabela dGeral}

\begin{itemize}
    \item \textbf{Mudança}: Deletar tabela \texttt{dGeral} (77 referências em medidas)
    \item \textbf{Medida testada}: \texttt{Projetos Jenkins}
    \item \textbf{Fase 1}: 89 objetos impactados identificados, com \texttt{requires\_manual\_review=True}
    \item \textbf{Fase 2}: aplicação automática corretamente bloqueada
    \item \textbf{Resultado}: Correto (bloqueio esperado)
\end{itemize}

\section{Disponibilidade do Dataset}
\label{sec:disponibilidade-dataset}

O código-fonte do agente e o script de execução do experimento estão disponíveis no repositório do projeto:

\begin{itemize}
    \item \textbf{Script de execução}: \texttt{experiments/executar\_experimento.py}
\end{itemize}

O repositório está publicado em:

\url{https://github.com/isabellasmou/agent-predict-power-bi}

O arquivo \texttt{.pbit} de origem e o JSON de resultado completo (\texttt{data/experimento\_completo\_resultados.json}) não são versionados no repositório público, por conter dados estruturais do modelo semântico utilizado; ambos podem ser obtidos sob solicitação ao autor ou reproduzidos integralmente a partir do código-fonte e do procedimento descrito na Seção a seguir.

\section{Reprodutibilidade}

Para reproduzir o experimento:

\begin{verbatim}
# Clonar repositorio
git clone https://github.com/isabellasmou/agent-predict-power-bi.git
cd agent-predict-power-bi

# Instalar dependencias
pip install -e ".[dev]"

# Configurar API key (Groq e gratuito)
echo "GROQ_API_KEY=sua_chave_aqui" > .env

# Colocar o arquivo .pbit na pasta data/

# Executar experimento (acha o .pbit automaticamente)
python experiments/executar_experimento.py
\end{verbatim}

Os resultados são salvos em \texttt{data/experimento\_completo\_resultados.json}, com métricas separadas por fase e por tipo de mudança.
