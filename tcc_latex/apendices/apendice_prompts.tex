\chapter{Prompts Completos Utilizados no Experimento}
\label{ap:prompts}

Este apêndice apresenta os prompts completos utilizados no experimento controlado (OE6), incluindo o system prompt e os templates de task prompts para cada tipo de refatoração.

\section{System Prompt (Versão 2.1)}

O system prompt define o papel do LLM e estabelece regras rígidas de comportamento:

\begin{lstlisting}[language=Python, caption=System Prompt v2.1, label=lst:system_prompt]
SYSTEM_PROMPT = """
Você é um especialista em DAX (Data Analysis Expressions) e Power BI com mais de 10 anos de experiência em refatoração de modelos semânticos.

Sua função é refatorar expressões DAX de forma precisa, preservando a lógica de negócio e seguindo as melhores práticas da linguagem.

REGRAS OBRIGATÓRIAS:
1. Retorne APENAS o código DAX refatorado, sem explicações
2. Use a sintaxe exata do DAX, incluindo aspas e colchetes conforme necessário
3. Preserve toda a lógica original — apenas atualize referências ao objeto renomeado
4. Não adicione aspas simples em torno de nomes de tabela/coluna, exceto quando o nome contiver espaços ou caracteres especiais
5. Mantenha a formatação e indentação original quando possível
6. Envolva o código DAX em tags XML: <dax>seu_codigo_aqui</dax>

IMPORTANTE: Evite alterar nomes de funções, operadores ou qualquer outro elemento que não seja a referência ao objeto renomeado.
"""
\end{lstlisting}

\section{Template 1: Renomeio de Coluna (v2.1)}

Template utilizado para casos C1-01 a C1-07:

\begin{lstlisting}[language=Python, caption=Template de Renomeio de Coluna, label=lst:prompt_column]
RENAME_COLUMN_PROMPT = """
TAREFA: Refatorar expressão DAX após renomeio de coluna.

CONTEXTO:
- Tabela: {table_name}
- Coluna ANTIGA: {old_name}
- Coluna NOVA: {new_name}
- Objeto impactado: {object_name} ({object_type})
{description_line}

EXPRESSÃO ORIGINAL:
{expression}

INSTRUÇÕES:
1. Substitua TODAS as referências a '{table_name}'[{old_name}] por '{table_name}'[{new_name}]
2. Se a sintaxe for {table_name}[{old_name}] (sem aspas na tabela), mantenha assim
3. Preserve exatamente a formatação, indentação e quebras de linha
4. NÃO altere funções DAX, operadores ou outras colunas
5. Retorne APENAS o código DAX refatorado em tags <dax>...</dax>

EXEMPLO de substituição correta:
- Original: SUM(Sales[Amount])
- Correto:  SUM(Sales[TotalAmount])
- ERRADO:   SUM('Sales'[TotalAmount])  ← NÃO adicione aspas desnecessárias!
"""
\end{lstlisting}

- Tabela NOVA: {new_table_name}

{expression}

1. Substitua TODAS as referências à tabela '{old_table_name}' por '{new_table_name}'

EXEMPLO:

- Correto:  CALCULATE(SUM('SalesData'[Amount]), 'SalesData'[Region] = "North")

\section{Exemplo de Prompt Completo (Caso C1-03)}

Abaixo, um exemplo de prompt completo real enviado ao LLM durante o experimento:

\begin{lstlisting}[language=Python, caption=Exemplo de Prompt Completo (Caso C1-03), label=lst:prompt_example]
System Prompt:
Você é um especialista em DAX (Data Analysis Expressions) e Power BI...
[System prompt completo acima]

User Prompt:
TAREFA: Refatorar expressão DAX após renomeio de coluna.

CONTEXTO:
- Tabela: Sales
- Coluna ANTIGA: Amount
- Coluna NOVA: SalesAmount
- Objeto impactado: Total Weighted (measure)
- Descrição: Medida com SUMX iterando sobre a coluna renomeada

EXPRESSÃO ORIGINAL:
SUMX(Sales, Sales[Amount] * Sales[Quantity])

INSTRUÇÕES:
1. Substitua TODAS as referências a 'Sales'[Amount] por 'Sales'[SalesAmount]
2. Se a sintaxe for Sales[Amount] (sem aspas na tabela), mantenha assim
3. Preserve exatamente a formatação, indentação e quebras de linha
4. NÃO altere funções DAX, operadores ou outras colunas
5. Retorne APENAS o código DAX refatorado em tags <dax>...</dax>

EXEMPLO de substituição correta:
- Original: SUM(Sales[Amount])
- Correto:  SUM(Sales[SalesAmount])
- ERRADO:   SUM('Sales'[SalesAmount])  ← NÃO adicione aspas desnecessárias!
\end{lstlisting}

\textbf{Resposta do LLM (Groq Llama 3.3 70B)}:

\begin{verbatim}
<dax>SUMX(Sales, Sales[SalesAmount] * Sales[Quantity])</dax>
\end{verbatim}

\textbf{Status}: Correto (acurácia exata = 100\%)

\section{Observações sobre Evolução dos Prompts}

A versão 2.1 dos prompts incorporou melhorias baseadas em falhas observadas na versão 2.0:

\begin{itemize}
    \item \textbf{Exemplo explícito de erro}: A instrução ``ERRADO: SUM('Sales'[TotalAmount])'' reduziu em 66\% os casos de aspas desnecessárias.
    \item \textbf{Formatação hierárquica}: Separar contexto, expressão e instruções melhorou a compreensão do LLM.
    \item \textbf{Tags XML}: O uso de \texttt{<dax>...</dax>} garantiu parsing 100\% confiável, eliminando erros de extração.
\end{itemize}

Detalhes completos sobre a evolução dos prompts estão documentados em \texttt{docs/PROMPT\_ENGINEERING.md} no repositório do projeto.
